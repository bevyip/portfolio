Flow Builder Canvas Enhancements

A case study about redesigning Salesforce's Flow Builder canvas layout and experience with improvements that are inclusive to those that are sight-impaired.

Due to a Non-Disclosure Agreement, information on this project is limited.

Introduction


Flow Builder provides declarative process automation for every Salesforce app, experience, and portal with point-and-click automation. I joined this team in April 2022 and have since been working on releasing changes that aim to consistently improve user productivity and experience.

The primary goal of this Flow Builder's redesign was to offer users a modernized and simplified layout that remains inclusive to all. This involves a cleaner and more visually appealing interface with improved navigation, as well as improving user experience for those that are sight-impaired with colors that meet the WCAG 2.1 requirements.

The modifications were implemented in Java, Typescript, HTML, and CSS.


So, what's the problem?


As the Flow Builder user base expands and increasingly complex flows emerge, we received feedback about the challenges of navigating the flow. Previously, the sole navigation method was dragging the canvas to view off-screen elements and scrolling to zoom for clearer visibility.

The following are the core problems this product aims to solve:

  • Users would lose element positioning on the canvas. The zoom level would reset with every element interaction. If you were building a large flow, this was a headache to deal with. It frustrated users and negatively impacted user productivity.
  • The icon colors were inaccessible. They did not meet the contrast ratio of at least 3:1 for graphics and user interface components. This was a requirement specified by Web Content Accessibility Guidelines (WCAG) 2.1. Our features were isolating our users, especially those that are sight-impaired.
